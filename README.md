# Today I Learn 페이지 만들기

## 생각해 보기
### HTML
나만의 갤러리에서 사진 9장은 HTML 태그 중 어떤 요소로 표현하는 것이 좋을까? 이유는 뭐라고 생각했는가?
- img tag를 사용하는 것이 좋을 거 같다. 

뼈대 코드를 기반으로 이미지를 로드하는 방식은 크게 2가지로 나뉘는데 무엇이고 어떻게 사용하고 있는가?
- img태그나 css의 background-image가 있다. img 태그는 문서 내부의 사진 등 컨텐츠와 관련된 데이터를 사용하고, background-image는 컨텐츠와 관계없는 사진으로 활용된다.

시맨틱 태그(예: <header>, <main>, <section>, <footer> 등)로 문서를 구분할 때, 어떤 기준으로 섹션을 나누고 태그를 적용하는 것이 좋을까?
- 컨텐츠의 의미와 역할을 기준으로 나누는 것이 좋을 것 같다.

<img> 태그의 alt 속성은 왜 필요하며, 어떻게 작성하는 것이 좋을까?
- 이미지가 무엇을 설명하는지에 대한 메타 정보로 활용될 수 있다. 최대한 직관적으로 이미지를 설명하는 게 좋을 것 같다.

이미지를 로드할 때, 상대 경로(relative path)와 절대 경로(absolute path)를 혼용하면 어떤 문제가 발생할 수 있을까?
- 해당 파일이 실행되는 환경이 바뀌면 절대 경로도 바뀌기 때문에 이미지가 깨질 수 있으며, 혼용될 경우, 상대 경로로 설정된 이미지는 보이고 절대경로의 이미지는 안 보이는 문제가 발생할 수 있다.

<meta> 태그의 viewport 설정과 Open Graph 태그는 각각 어떤 역할을 하는가?
- viewport는 모바일 기기에서 화면 크기에 맞추어 비율이 설정되는 것을 돕는다.
- 웹페이지가 링크 형태로 공유될 때 미리보기로 제공되는 정보를 설정한다.


### CSS
나만의 갤러리에서 9장의 이미지를 보기 좋게 배치하려면, Flexbox와 CSS Grid 중 어떤 것이 더 적합할까? 그 이유는?
- Grid가 적당한 것 같다. Flex는 1차원으로 배치되었기 때문에 사진을 한 눈으로 파악하기 어려웠기 때문이다.

태그 선택자, 클래스 선택자, ID 선택자는 각각 언제 사용하는 것이 적절할까?
- 태그 선택자는 문서의 기본적인 스타일 등을 잡을 때 사용한다.
- 클래스 선택자는 동일한 스타일을 여러 요소에 재사용할 때 사용한다.
- ID 선택자는 페이지 내에서 한 번만 사용되는 요소에 적용한다. 주로 JS로 기능을 동작시킬 때 사용한다.

CSS를 HTML 내에 직접 작성하는 것과 별도 파일로 분리하는 것의 차이는 무엇일까?
- 브라우저가 CSS 파일을 캐싱할 수 있어 반복되는 CSS 파일 다운로드를 방지할 수 있다.
- HTML과 CSS가 구분되어 유지보수가 쉬워진다.

### JavaScript
document.getElementById()와 document.querySelector()의 차이는 무엇인가?
- getElementById는 id를 기반으로 탐색한다. querySelector()는 CSS 선택자를 사용하여 탐색이 유연하고 강력하다.

이벤트 리스너를 사용해서 "어떤 일이 일어났을 때 어떤 동작을 실행"하게 만들 수 있다. 자기소개 페이지에서 활용할 수 있는 이벤트에는 어떤 것들이 있을까?
- 취미를 눌렀을 때 TMI가 모달형태로 나오게 하거나, TIL에 입력된 문장에 금지어가 포함된 경우 업로드가 안 되게 할 수도 있을 것 같다.

innerHTML로 DOM을 조작할 때 주의해야 할 점은 무엇일까?
- innerHTML은 웹 페이지 요소 안에 HTML 코드를 사용할 수 있게 하는 것이다. 
- DOM은 작성된 HTML 코드를 트리 형태의 구조로 변환된 것으로, <html>, <body> 등의 태그들은 노드가 된다. 
- innerHTML로 삽입된 문자열을 특정 요소에 넣으면 브라우저는 기존 DOM 노드를 삭제한다. 그리고 다시 삽입된 문자열을 포함한 가지를 만들어 포함시킨다.
- 이때 악성 스크립트를 HTML로 삽입하면 웹페이지가 의도하지 않은 동작을 수행해 보안에 문제가 생길 수 있다.

### 배포
GitHub Pages로 배포한 페이지에서 TIL을 작성하면, 새로고침하거나 재배포했을 때 데이터가 유지될까? 그 이유는?
- 반영되지 않는다. 현재 JS로 추가되는 로직이 수행되지만, 로컬 브라우저에 반영되었을뿐, 그 변동사항이 저장될 공간이 없기 때문이다.

JavaScript로 추가한 데이터를 영구적으로 저장하려면 어떤 방법이 있을까?
- 어떤 형태로든 DB를 도입하여 TIL 등록 버튼을 누르면 DB에 데이터가 저장되도록 한다. 그리고 데이터가 반영되면 페이지에서 자동 새로고침하여 변동사항을 반영한다.
